# Projects

## Smart Office Face Recognition System
Mihir's Kingston CI7523 project is a real-time computer-vision access-control prototype built around webcam input. The pipeline combines OpenCV preprocessing, face detection with MTCNN and RetinaFace, and recognition using FaceNet and ArcFace embeddings. The CV extract emphasizes practical robustness work: CLAHE and Gaussian denoising to improve varied-lighting conditions, migration from TensorFlow MTCNN to facenet-pytorch, and standardizing the embedding database to consistent 512D vectors to avoid silent match failures. Mihir wrapped the app in Streamlit, with caching used to reduce rerender overhead and model reloads. What makes the project strong is that it is not just a model demo; it is a complete pipeline with enrollment, thresholding, unknown-face rejection, and a reproducible interface that can be shown to reviewers. It demonstrates both computer-vision depth and deployment awareness.

## Multi-Class Classification on Imbalanced Data
Mihir's Kingston CI7521 CW1 project benchmarks eight classical classifiers on a five-class imbalanced dataset. The goal was not to chase one model blindly, but to establish a fair and reproducible comparison across LDA, QDA, Decision Tree, KNN, Logistic Regression, SVM, Random Forest, and Gaussian Naive Bayes. The work used a quantile transformer to reduce skew, SMOTE oversampling inside cross-validation folds to avoid leakage, and a two-stage hyperparameter search with RandomizedSearchCV followed by GridSearchCV. Evaluation went beyond raw accuracy and used balanced accuracy, macro and micro one-vs-rest ROC-AUC, and confusion matrices, with macro-F1 used as the main ranking metric. The project is valuable because it shows disciplined experimental design, leakage avoidance, and careful interpretation of class-imbalanced results rather than only a final leaderboard.

## Medical Image Classification — OrganSMNIST
Mihir's Kingston CI7521 CW2 project studies abdominal CT organ classification using OrganSMNIST at 224x224 resolution. Mihir ran 13 controlled experiments across MLPs, custom CNNs, and transfer-learning approaches using MobileNetV2 and ResNet50. The key idea was to compare families of models under consistent splits, augmentation, and training settings so that the outcome reflected architecture choice rather than experiment noise. The CV extract notes a baseline MLP, deeper variants with dropout and batch norm, SGD with momentum, L2 regularization, AdamW, and a data-augmentation variant that reduced overfitting. ResNet50 produced the strongest generalization on the held-out test set. This project shows structured experimentation, a solid grasp of computer-vision training regimes, and comfort with reporting metrics such as accuracy, F1, precision, recall, and confusion matrices in a medical-imaging context.

## Growatt CRM — InfoSystem Dashboard
Built during Mihir's NTech Infoway internship, this MERN-stack CRM targeted Growatt InfoSystem’s need to manage clients, service histories, and field-engineer workloads in one place. The portfolio summary describes a React frontend, REST APIs with Node.js and Express, MongoDB schemas, and role-based views for admin and field-engineer users. The application addressed a real operational pain point: the team was previously relying on spreadsheets and paper, which made it difficult to keep a single, accurate source of truth. The project’s strength is in its product thinking. It includes protected routes, JWT auth with role claims, mobile-friendly views for engineers in the field, reusable dashboard components, and a shared validation layer between UI and API. The outcome was a production CRM that replaced fragmented workflows with something maintainable and role-aware, which is the kind of work that translates well into any full-stack role.

## HappinessFactors Website
Mihir's freelance project delivered a Laravel company website for HappinessFactors. The site used Blade templates and Tailwind CSS for the front end, with Groove.cm integrated as a headless content-management layer so a non-technical team could update pages, posts, and media without touching code. The key problem was not building a pretty brochure site; it was making the site maintainable for the client after handoff. The project therefore focused on server-rendered pages, reusable partials, content-model mapping, and caching choices that balanced freshness with performance. The CV also highlights digital-content work alongside the technical delivery, which means the role was partly engineering and partly operational support for marketing output. The result was a live site that the client can maintain independently, which is a strong signal of practical delivery and not just implementation skill.
